# Apple / Ball Trajectory Prediction

Проект определяет центр яблока или теннисного мяча по сегментации YOLO и
прогнозирует его будущую **2D-точку на изображении**. Основной горизонт
прогноза задаётся `PREDICT_DELTA` (по умолчанию 80 мс).

## Запуск

```bash
python3 -m pip install -r requirements.txt
python3 main.py
```

`main.py` открывает камеру, загружает веса
`best_apple_colab_30epochs.pt` и показывает центр сегментации, прогноз и
метрики. `q` завершает программу, `m` включает/скрывает показатели.

Для воспроизводимого эксперимента без камеры:

```bash
python3 main_mock.py
```

В mock-окне доступны сценарии:

- `1` — равномерное движение;
- `2` — свободный баллистический бросок;
- `3` — затухающий маятник;
- `4` — отскоки в замкнутом ящике;
- `5` — свободный полёт с ударом о вертикальную стену.

## Как считается ошибка

Прогноз, созданный в момент кадра `t`, проверяется в момент `t + Δ`, а не на
следующем произвольном кадре. Если цель лежит между двумя надёжными
измерениями, истинная координата интерполируется. Прогнозы во время длинной
потери объекта не смешиваются с новой траекторией и отмечаются как
пропущенные.

В реальном режиме «истина» — это следующий качественный центр сегментации,
поэтому метрика включает как ошибку прогноза, так и шум детектора. В mock
режиме предиктор получает шумное наблюдение, а ошибка вычисляется по известной
физической позиции объекта. Это позволяет отделить качество предиктора от
ошибки сегментации.

В отчёте выводятся средняя и медианная 2D-ошибка, RMSE, P95, максимум, bias по
осям и доля прогнозов в заданном пороге. Все величины пока в пикселях.

## Настройка

Главные параметры находятся в `config.py`:

- `OBJECT_PROFILE` — `apple`, `tennis_ball` или `tennis_ball_alt`; выбирает
  веса модели и имя класса сегментации;
- `PREDICT_DELTA` — горизонт прогноза;
- `MIN_CONFIDENCE` — минимальная уверенность сегментации;
- `MAX_SPEED_PX_PER_SEC` и `JUMP_MARGIN_PX` — фильтр невозможных скачков;
- `MAX_EVALUATION_GAP` — максимальный разрыв кадров для честной оценки;
- `GRAVITY_PX` — гравитация в mock-физике.

Параметры `KALMAN_MEASUREMENT_STD_PX`, `KALMAN_JERK_NOISE` и
`KALMAN_INNOVATION_GATE` настраивают соответственно доверие к центру
сегментации, реакцию на манёвры и отсев статистически невероятных измерений.

По умолчанию используется `kalman_ca`: фильтр Калмана с постоянным ускорением.
Он хорошо подходит для короткого свободного полёта. `poly2` может быть полезен
как эталон для чистой параболы, но менее устойчив к шуму сегментации.

Mock по умолчанию использует честный базовый `kalman_ca`: он не знает о стене
до столкновения. `collision_ca` — отдельный эксперимент: он отражает скорость
только от явно заданных стен и нужен для проверки модели с известной
геометрией. В реальном режиме выберите `PREDICTOR_NAME = "collision_ca"`,
включите `COLLISION_REAL_ENABLED` и заполните `COLLISION_REAL_WALLS` лишь
после калибровки реальной стены в координатах кадра.

## Тесты

```bash
python3 -m unittest discover -v
```

Тесты проверяют геометрию центра сегментации, временное сопоставление прогноза
с истиной, защиту от выбросов/дублированных timestamp, Kalman на разных FPS и
физику mock-сценариев.

## Ограничения

С одной камерой проект предсказывает координату в пикселях, а не 3D-положение
или расстояние в сантиметрах. Для метрической ошибки нужна калибровка камеры и
известная плоскость движения либо дополнительный источник глубины. Обычный
Kalman не может предсказать отскок от неизвестной стены до первого контакта:
для этого нужен отдельный collision-aware predictor с геометрией стены,
радиусом мяча и коэффициентом реституции.
